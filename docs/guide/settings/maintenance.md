# 维护

日常维护和问题排查相关的功能。

## 任务历史

进入路径：`设置` → `其他` → `任务历史`。

任务历史记录所有批量操作的执行结果，包括批量匹配、批量编辑、批量重命名、格式化歌词、计算回放增益、导出歌词/封面等。

### 查看与筛选

- 按 **"任务类型"** 筛选：批量匹配、格式化歌词、导出歌词等。
- 按 **"状态"** 筛选：排队中、运行中、成功、失败、跳过、已取消。

### 管理任务

- 运行中或排队中的任务可以 **取消**。
- 已结束的任务记录可以 **删除**。
- "清空历史"只删除当前筛选条件下已结束的任务记录，运行中的任务会保留。

::: warning 删除任务记录不会撤销修改
删除任务记录不会撤销已经写入音频文件的标签修改。它只是清理历史记录。
:::

### 查看任务详情

进入某个任务，可在"成功"、"失败"、"跳过"标签页查看每首歌的处理结果。即使任务整体显示"成功"，仍可能有部分歌曲失败或跳过。

## 应用日志

进入路径：`设置` → `其他` → `应用日志`。

应用日志用于排查问题，覆盖以下类型：

- 崩溃与异常
- 元数据读写
- 批量操作
- 数据库
- 网络请求
- 插件执行

### 使用日志

- 按 **"级别"** 筛选：DEBUG、INFO、WARN、ERROR。
- 按 **"类型"** 筛选：插件、网络、崩溃、批量等。
- 点击日志条目查看详细信息。
- **复制**单条日志或 **导出**日志文件。

### 日志保留周期

可选：
- 不记录
- 7 天（默认）
- 30 天
- 90 天
- 永久保留

默认的 7 天适合日常排错。长期保留会占用更多存储空间。

## 清理缓存

"清理缓存"会统计并清理：
- 图片缓存（搜索封面时下载的图片）
- 网络缓存
- 外部缓存
- 其他临时文件

清理缓存不会删除音乐文件，也不会清除已写入音频文件的标签。

如果经常搜索封面，或将"搜索限制数"设置得较大，图片缓存可能增长较快。建议定期清理。
