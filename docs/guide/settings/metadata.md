# 元数据处理

控制元数据文本处理和显示相关的设置。

## 中文文本转换

"中文文本转换"会应用于搜索到的文本元数据和歌词。ID、日期、数字、URL 和封面不会受转换影响。

可选模式：
- 不转换
- 简体转繁体
- 繁体转简体

## 移除空行

开启后，保存歌词和其他多行文本字段时会自动删除其中的空行。

## 非歌词内容过滤

用于在"格式化歌词"中勾选"移除非歌词内容"时，自动清理作词、作曲、来源等非歌词行。

过滤规则在 `设置` → `元数据处理` → `非歌词内容过滤规则` 中维护。规则按完整文本匹配原始歌词行；逐字歌词会先拼成整行后再判断。

常见过滤示例：`作词 :`、`作曲 :`、`编曲 :`、`来源 QQ音乐` 等。

## 艺术家分隔符

多个艺术家写入标签时使用的分隔符。默认分隔符在不同音频格式中可能不同（如 FLAC 使用 `\0` 分隔符，MP3 使用 `; ` 等）。

## 艺术家拆分

**"艺术家拆分规则"** 影响艺术家视图的归类方式。开启后，Lyrico 会将包含多个艺术家的歌曲分别在艺术家中归类。

可以在拆分设置中配置：
- 额外的分隔符（如 `feat.`、`&`、`,` 等）。
- **不拆分明细**：某些艺术家名称不应被拆分（如艺术家名本身就包含分隔符的情况）。

修改分隔符或白名单后，通常需要点击 **"重建艺术家索引"** 才会完全生效。

## 字段显示设置

控制单曲编辑页和批量编辑页中 **显示哪些标签字段**。

被隐藏的字段：
- 不会出现在编辑界面中。
- 在保存时 **不会被清空**（已有的标签数据保留不变）。

适用于简化编辑界面，隐藏不常用的字段。

## 自定义标签管理

控制哪些自定义标签 key 显示在编辑页和批量编辑页中。

- 添加一个自定义标签 key 后，它会在编辑页的自定义标签区域显示。
- 移除显示不会删除歌曲中已有的自定义标签数据，只是不在编辑界面显示。
- 系统会跟踪每个自定义标签 key 的使用次数。

## 配置字符映射

用于批量重命名时，将文件名中的特定字符替换为其他字符。

在字符映射页中，点击字符可修改其替换规则。每个字符可以映射到不同的替换值。常见用法例如将 `/` 替换为 `-`，或将 `:` 替换为 `_` 等不适合作为文件名的字符。
